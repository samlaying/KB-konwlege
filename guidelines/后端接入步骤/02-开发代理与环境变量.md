# 步骤 2：开发代理与环境变量

目标：让前端开发环境可以稳定访问本地后端。

## 需要改造

- 在 [`F:\samlay\KB-konwlege\vite.config.ts`](F:\samlay\KB-konwlege\vite.config.ts) 中增加 `/api` 代理
- 增加环境变量：
  - `VITE_API_BASE_URL`
  - `VITE_SSE_BASE_URL`

## 当前代码现状

- [`F:\samlay\KB-konwlege\vite.config.ts`](F:\samlay\KB-konwlege\vite.config.ts) 还没有代理配置
- 当前页面代码还没有任何环境变量读取逻辑
- [`F:\samlay\KB-konwlege\src\app\services\api.ts`](F:\samlay\KB-konwlege\src\app\services\api.ts) 为空，因此这一步要为后续 API 层先准备好运行环境

## 建议新增的文件

- `F:\samlay\KB-konwlege\.env.example`
- `F:\samlay\KB-konwlege\.env.development`
- `F:\samlay\KB-konwlege\.env.production`

## 建议环境变量取值

- `VITE_API_BASE_URL=/api`
- `VITE_SSE_BASE_URL=/api`
- 如果后端 SSE 单独走另一域名，再把 `VITE_SSE_BASE_URL` 切到真实地址

## Vite 代理建议

- `/api` 代理到 `http://localhost:8000`
- 先只代理 API，不要把全部静态资源也走代理
- 如果后端 SSE 也是同域 `/api/chat`，通常不需要单独代理规则

## 这一阶段的落地要求

- 组件和 store 内部不得出现硬编码 `http://localhost:8000`
- API 层统一从 `import.meta.env` 读取地址
- 如果后端暂时没启动，前端至少能明确报网络错误，而不是静默失败

## 建议

- 本地开发走 Vite 代理，避免跨域
- 生产环境直接读取环境变量，不依赖代理
- 组件内部不要写死 `http://localhost:8000`

## 完成标准

- 浏览器访问前端时，请求 `/api/*` 能转发到后端
- API 地址可通过环境变量切换
- 文档里已经明确哪些地址用于普通 HTTP，哪些用于 SSE
