# 实施顺序与改造文件清单

## 推荐实施顺序

1. 接口契约
2. 代理和环境变量
3. API 层
4. Zustand 异步化
5. 主题和设置
6. 文档上传和索引
7. 聊天和 SSE
8. 历史记录、异常处理、验收

## 每一步的主要产出

1. 接口契约：定 DTO、定 SSE 事件、定错误返回格式
2. 代理和环境变量：能在本地稳定请求 `/api`
3. API 层：提供 `themeApi`、`documentApi`、`conversationApi`、`settingsApi`、`chatApi`
4. Zustand 异步化：移除 Mock，补 `loading/error`
5. 主题和设置：先跑通最简单的 CRUD 和保存链路
6. 文档上传和索引：跑通上传、轮询、删除
7. 聊天和 SSE：跑通真实问答和来源展示
8. 历史记录、异常处理、验收：补齐可用性
