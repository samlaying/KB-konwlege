# 步骤 6：接入文档上传和索引状态

目标：让知识库管理页真正能上传文档、导入网页、添加文本，并观察索引过程。

## 涉及页面

- `src/app/pages/DocumentsPage.tsx`

## 当前代码现状

- [`F:\samlay\KB-konwlege\src\app\pages\DocumentsPage.tsx`](F:\samlay\KB-konwlege\src\app\pages\DocumentsPage.tsx) 通过 `beforeUpload={() => false}` 阻止真实上传
- `handleSubmit()` 里直接拼出一个前端文档对象，然后用两层 `setTimeout` 模拟索引状态
- 删除文档也只是删本地 store，没有请求后端

## 需要拆开的三种导入路径

- 文件上传：`POST /api/documents/upload`
- 网页导入：`POST /api/documents/url`
- 文本导入：`POST /api/documents/text`

## 文件上传阶段的具体要求

- 用 `FormData` 传 `file` 和 `themeId`
- `Upload` 组件继续保留，但不再只做本地文件选择
- 上传成功后把返回的文档写入 store
- 如果后端立即返回 `pending` 状态，则前端立刻开始轮询

## 状态轮询建议

- 轮询接口：`GET /api/documents/:id/status`
- 轮询间隔：2 到 3 秒
- 停止条件：`done` 或 `failed`
- 失败时记录 `errorMessage`
- 页面卸载时清理轮询定时器

## 建议改造点

- 把 `handleSubmit()` 拆成三个 action 分支
- 筛选主题逻辑可以保留，但数据源改为后端列表
- 表格 `rowKey="id"` 可以继续使用，前提是后端返回稳定 ID
- 如果后端支持分页，这里后面可以再补；第一版先全量加载即可

## 需要改造

- 去掉 `beforeUpload={() => false}` 的纯前端拦截逻辑
- 文件上传使用 `FormData`
- 网页导入调用 `POST /api/documents/url`
- 文本导入调用 `POST /api/documents/text`
- 上传完成后启动状态轮询 `GET /api/documents/:id/status`

## 建议补充能力

- 上传进度条
- 状态轮询间隔，例如 2 到 3 秒
- 状态失败时展示错误信息
- 文档删除后同步刷新列表和主题统计
- 上传完成后自动高亮或滚动到新文档，可作为增强项

## 完成标准

- 上传文件后后端能收到真实文件
- 文档状态会从 `pending` 进入 `indexing` 再到 `done/failed`
- 删除文档后页面和主题统计都与后端一致
